# SPEC.md - 视频转精灵图工具 v3.0 节点交互重构

## 目标

重构节点工作流编辑器，将「双击弹窗编辑」改为「单击原位展开编辑」，支持节点自由添加/删除/连线，明确数据类型体系，改善 AI 抠图的逐帧进度反馈。

## 目标用户

独立游戏开发者、动画师、像素画创作者，需要从视频/图片制作精灵图素材。

## 核心变更（相对 v2.0）

| 变更项 | v2.0（当前） | v3.0（目标） |
|--------|-------------|-------------|
| 节点编辑 | 双击打开侧边抽屉面板 | 单击节点原位展开内嵌 UI |
| 节点管理 | 9 个固定节点，不可增删 | 右键菜单添加，可删除任意节点 |
| 连线操作 | 从输出端口拖拽创建 | 拖拽创建 + 右键端口断开 + 拖拽已有连线重连 |
| AI 抠图进度 | 进度条 + 计数 | 逐帧缩略图实时显示 |
| 数据类型 | 隐式，靠端口 label | 显式类型系统，端口颜色标注 |

## 核心功能与验收标准

### F1. 节点内嵌 UI（替代弹窗面板）

**删除**：`detail-overlay`、`detail-panel`、`DP`（Detail Panel）对象、所有 `buildDetailHTML` / `bindDetailEvents` 相关代码。

**新增**：节点在画布上原位展开，直接在 `.wf-node-body` 内渲染完整操作 UI。

- 节点默认折叠态：仅显示标题栏 + 端口 + 一行状态摘要（与当前一致）
- 单击节点标题栏：展开节点体，显示该节点类型的完整操作 UI
- 再次单击或点击其他节点：折叠当前节点
- 展开后节点宽度从 200px 过渡到合适尺寸（约 420-480px），确保 UI 可用
- 同一时刻最多一个节点展开（类似 radio group 行为）

**展开后节点内部布局**：
```
+------------------------------------------+
| icon  节点标题                     折叠按钮 |
+------------------------------------------+
| [操作 UI 区域]                            |
|  - 输入框、按钮、滑块、预览等               |
|  - 可滚动（max-height 限制）               |
+------------------------------------------+
| [输入端口]              [输出端口]         |
+------------------------------------------+
| [删除节点按钮]                            |
+------------------------------------------+
```

**验收**：单击节点展开，UI 可操作，再次单击或点其他节点折叠。无弹窗出现。

### F2. 节点添加与删除

#### F2a. 右键菜单添加节点

- 在画布空白处右键：弹出节点类型菜单
- 菜单列出所有可用节点类型（9 种）
- 点击菜单项：在右键位置创建该类型的新节点
- 新节点自动获得唯一 ID（如 `aiImage_2`、`removeBg_3`）

#### F2b. 节点删除

- 每个展开的节点底部显示「删除节点」按钮（红色次要样式）
- 删除节点时：该节点相关的所有连线一并删除，SVG 元素移除
- 删除前 toast 确认提示（非 confirm 弹窗）

**验收**：右键画布可添加节点，删除节点后连线消失，画布正常。

### F3. 连线操作改进

#### F3a. 创建连线（保留现有）

- 从输出端口圆点拖拽到输入端口圆点，创建连线
- 类型兼容性检查不变

#### F3b. 断开连线（新增）

- 右键点击输入端口：弹出该端口的所有连线列表，点击可断开
- 或：右键点击连线本身，菜单选「断开」

#### F3c. 拖拽重连（新增）

- 按住已有连线的输出端圆点拖拽：断开该连线，进入拖拽创建模式，可连到新的输入端口

**验收**：右键端口可断开连线，拖拽端口可重连，操作直观。

### F4. 数据类型体系

| 类型 | 颜色 | 格式说明 | 兼容输入 |
|------|------|---------|---------|
| `image` | `#63b3ed`（蓝） | `{ urls: string[] }` 图片 URL 数组 | image, frames, sprite |
| `video` | `#a78bfa`（紫） | `{ url: string, blobUrl?: string }` 视频 | video |
| `frames` | `#34d399`（绿） | `{ frames: Frame[] }` 帧对象数组，每帧含 dataUrl | image |
| `sprite` | `#fbbf24`（橙） | `{ dataUrl: string, frames: Meta[], meta: object }` 精灵图 | image |

**类型转换规则**（数据通过连线时自动转换）：
- `frames -> image`：提取帧的 dataUrl 数组作为 `urls`
- `sprite -> image`：提取 sprite.dataUrl 作为 `urls[0]`
- `image -> frames`：将 urls 下载为帧对象数组

**验收**：连线时类型不兼容则拒绝，兼容时自动转换数据格式。

### F5. 节点定义

| 类型 ID | 名称 | 输入端口 | 输出端口 | 说明 |
|---------|------|---------|---------|------|
| `aiImage` | AI 生图 | `prompt`(image) | `images`(image) | 文生图/图生图 |
| `aiVideo` | AI 生视频 | `image`(image) | `video`(video) | 从首帧生成视频 |
| `importVideo` | 导入视频 | `video`(video) | `video`(video) | 上传/URL/接收视频 |
| `selectFrames` | 选择帧 | `video`(video) | `frames`(frames) | 视频拆帧 + 选帧 |
| `directSprite` | 直接合成 | `frames`(frames) | `sprite`(sprite) | 跳过抠图直接合成 |
| `img2img` | 图生图 | `images`(image) | `images_out`(image) | AI 图生图 |
| `removeBg` | 抠图 | `images`(image) | `processed`(image) | AI/颜色抠图 |
| `composeSprite` | 合成精灵图 | `processed`(image) | `sprite`(sprite) | 带透明背景合成 |
| `export` | 导出 | `sprite`(sprite) | 无 | PNG/JSON/GIF 导出 |

**节点工厂**：`NODES` 数组变为「节点类型模板」。创建新实例时从中复制定义，生成唯一实例 ID。

### F6. AI 抠图逐帧反馈

当前问题：AI 抠图处理多帧时只显示进度条和计数，看不到每帧处理结果。

**改为**：在抠图节点展开 UI 中，处理过程中实时显示已完成的帧缩略图网格。

- 每处理完一帧：在预览区域追加一个缩略图（带帧号标签）
- 缩略图使用处理后的 dataUrl 直接渲染，无需等待全部完成
- 进度条保留，显示在缩略图网格上方

**验收**：AI 抠图处理 10 帧时，能看到第 1、2、3... 帧依次出现在节点内。

### F7. 数据缓存（保留 v2.0 修复）

每个节点的处理结果缓存在全局 `S` 状态和 `NM.nodes[id].data` 中。节点折叠再展开时自动恢复已有数据。此功能已在 v2.0 中实现，v3.0 保留。

## 架构变更

### 删除的模块

- `DP`（Detail Panel）对象及相关 HTML（`detail-overlay`、`detail-panel`）
- `buildDetailHTML()` 函数
- `bindDetailEvents()` 函数
- 所有 `xxxHTML()` 函数（如 `aiImageHTML()`、`aiVideoHTML()`）

### 新增/重写的模块

- **NodeRenderer**：每个节点类型注册一个渲染函数，负责生成内嵌 UI HTML
- **NodeEventBinder**：每个节点类型注册一个事件绑定函数，负责交互逻辑
- **NodeFactory**：节点工厂，管理节点类型的创建和实例化
- **ContextMenu**：右键菜单管理（添加节点、断开连线）
- 连线管理增强：断开、重连逻辑

### 数据结构变更

```javascript
// 节点定义模板（类型，非实例）
const NODE_TYPES = [
  { typeId: 'aiImage', icon: '...', title: 'AI 生图', ins: [...], outs: [...], render: renderAiImage, bind: bindAiImage },
  ...
];

// 节点实例
NM.nodes[instanceId] = {
  el,           // DOM 元素
  typeId,       // 类型 ID（如 'aiImage'）
  collapsed,    // 是否折叠
  expanded,     // 是否展开（显示内嵌 UI）
  data,         // 上游推入的数据
  status,       // idle | processing | done
  instanceData  // 节点自身的缓存数据（生成结果等）
};
```

### 持久化

- 保存：节点实例列表（类型 + 位置 + 折叠状态）+ 连线列表
- 键名不变：`spritetool_wf_layout`
- 刷新后恢复布局和连线，处理数据不持久化

## 不做什么

- 不改变 server.js
- 不改变 AI API 调用逻辑
- 不做节点类型自定义/插件系统
- 不做多人协作
- 不做撤销/重做（后续可加）
- 不改变核心业务算法（抠图、像素画、GIF 编码）
